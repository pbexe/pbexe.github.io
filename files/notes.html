Research task

Find out how signed binary integers can be stored using 2’s compliment.

Two’s complement is a clever way of storing integers so that common math problems are very simple to implement.

To understand, you have to think of the numbers in binary.

It basically says,

  • for zero, use all 0’s.
  • for positive integers, start counting up, with a maximum of 2(number of bits - 1)-1.
  • for negative integers, do exactly the same thing, but switch the role of 0’s and 1’s (so instead of starting with 0000, start with 1111 - that’s the “complement” part).

Let’s try it with a mini-byte of 4 bits (we’ll call it a [nibble][1] - 1/2 a byte).

  • 0000 - zero
  • 0001 - one
  • 0010 - two
  • 0011 - three
  • 0100 to 0111 - four to seven

That’s as far as we can go in positives. 23-1 = 7.

For negatives:

  • 1111 - negative one
  • 1110 - negative two
  • 1101 - negative three
  • 1100 to 1000 - negative four to negative eight

Note that you get one extra value for negatives (1000 = -8) that you don’t for positives. This is because 0000 is used for zero.

Doing this, the first bit gets the role of the “sign” bit, since it is always ‘1’ for negative numbers, and ‘0’ for non-negatives (zero and positive).

Information coding systems

The problem of storing text in a digital form pre-dates the computer era. A variety of different digital coding schemes have been used from prehistoric times (ie: beacon fires) to relay predetermined messages. Other examples include Morse code and semaphore systems.

With the introduction of computer systems in the 1940s and 50s it became clear that text as well as numbers would need to be stored and manipulated by computers and a number of systems for encoding text characters were developed. The best being known as ASCII (“The American standard code for information exchange”). This used 7 bits to encode the standard English language character set along with a number of punctuation and control characters and numbers 0-8.

ASCII has now been superseded by Unicode for most applications. Unicode uses as minimum of 8 bits but can use up to 48 bits. UTF-8 uses 8 bits for encoding Western text. Unicode has the advantage that it can encode most of the character sets used in the world.

Majority voting
One other way to detect and correct errors in a transmission is to send a message 3 or more times:

where each colour arrow shows a different message. If an error occurs it looks like this:

The error has only occurred in copy 2 so copies 1 and 3 are can be assumed to be correct as they are in the majority.
Broken down into individual steps:
  1. The first digit is 1 in all 3 copies so it must be correct.
  2. The second digit is 0 in copies 1 and 3 so majority voting concludes that is the correct number.
  3. The remaining digits are all the same in all the copies so they must be correct.
Majority voting is used most often is safety critical systems.

A check digit is a number added to a set of data that ensures that the data has been correctly received. Check digits are commonly used for bar-codes. A simple example may be a system that works by adding up the digits of a number and adding a check digit to make a standard total. The only problem with this is that it doesn’t check whether the numbers are out of order.

Abstract data structures

Abstract data structures are commonly used for data that is not specific to any one programming language. The simplest example is a stack. Stacks can be used to solve many problems, such as reversing a string and may also be used to implement features of a programming language.
Describing abstract data structures also helps us to understand another important programming concept: object orientation.

Stack

A stack is a data structure which can be accessed from only one end:

In python, a stack may be described using a class:

class Stack:
    def __init__(self):
        self.items = []
        
    def isEmpty(self):
        return self.items == []
    
    def push(self, item):
        self.items.append(item)
        
    def pop(self):
        self.items.pop()
        
    def peek(self):
        return self.items[len(self.items)-1]
    
    def size(self):
        return len(self.items)

To use this stack class, you use the following code:

myStack = Stack()           # Creates a new stack
print(myStack.isEmpty())    # Prints True because the stack is empty
myStack.push("Hello World") # Pushes "Hello World" to the stack
print(myStack.isEmpty())    # Prints False because the stack isn't empty
print(myStack.size())       # Prints 1 because there is one item in the stack
print(myStack.peek())       # Prints "Hello World" because it is at the top of the stack
Queues

A queue is an ordered collection of items where addition of new items occurs at the end and removal of items occurs at the head.
As items are added and removed from the queue, items already in the queue are moved towards the head.

Queues can be used to buffer data. For example in a multi-user printer system, where print jobs are submitted to a networked printer and then printed in order of arrival.
Another example is a keyboard buffer. Keystrokes are stored in order until they are processed. Streamed video is also buffered (obviously).

Queue operations

  1. Enqueue
    : Add an item to a queue

  2. Dequeue
    : Remove an item from a queue

  3. IsEmpty
    : Check to see if the queue is empty

  4. Size
    : Find the size of the queue

Python definition of a queue:

class Queue:
    def __init__(self):
        self.items = []
        
    def isEmpty(self):
        return self.items == []
    
    def enqueue(self, item):
        self.items.insert(0,item)
        
    def dequeue(self):
        return self.items.pop()
    
    def size(self):
        return len(self)

A graph can be expressed in python as the following class:

class Node:
    def __init__(self, key):
        self.id = key
        self.connectedTo = {}
    def addNeighbour(self, nbr, weight=0):
        self.connectedTo[nbr] = weight
    def __str__(self):
        return str(self.id) + " connected to " + str([x.id for x in self.connectedTo])
    def getConnections(self):
        return self.connectedTo.keys()
    def getId(self):
        return self.id
    def getWeight(self, nbr):
        return self.connectedTo[nbr]
class Graph:
    def __init__(self):
        self.nodeList = {}
        self.numNodes = 0
    def addNode(self, key):
        self.numNodes = self.numNodes + 1
        newNode = Node(key)
        self.nodeList[key] = newNode
        return newNode
    def getNode(self, n):
        if n in self.nodeList:
            return self.nodeList[n]
        else:
            return Node
    def __contains__(self, n):
        return n in self.nodeList
    def addEdge(self, f, t, cost=0):
        if f not in self.nodeList:
            nv = self.addNode(f)
        if t not in self.nodeList:
            nv = self.addNode(t)
        self.nodeList[f].addNeighbour(self.nodeList[t], cost)
    def getNodes(self):
        return self.nodeList.keys()
    def __iter__(self):
        return iter(self.nodeList.values())

Tree terminology

Nodes and edges are the same as graphs.

Note
Each node (except the root node) has one incoming node but one or more outgoing nodes.

Root
The only node that has no incomming edges.
Path
An ordered list of nodes connected by edges.
Children
Set of nodes that have incoming edges from the same node.
Parent
A node is the parent of all nodes that connect to it’s outgoing edges.
Sibling
Child nodes of the same parent nodes are called siblings.
Subtree
A subtree is a set of nodes and edges comprised of of a parent and all the descendants of that parent.
Leaf node
A node with no children.
Level
The level of a node is how many edges it is away from the root node.
Height
The height of a tree is equal to the maximum level of any node in the tree.
Definition of a tree

A tree consists of a set of nodes and a set of edges that connect pairs of nodes. A tree has the folowng properties:

  • One node of a tree is designated as the root node.
  • every node n, except the root node is connected by an edge to exactly one other node, where that node is the parent node.
  • A unique path traverses from each node ot teh root.
  • If each node has a maximum of 2 children, the tree is binary.

Another way of representing a tree is through object orientation:

class BinaryTree:
    def __init__(self, rootObj):
        self.key = rootObj
        self.leftChild = None
        self.rightChild = None
    def insertLeft(self, newNode):
        if self.leftChild == None:
            self.leftChild = BinaryTree(newNode)
        else:
            t = BinaryTree(newNode)
            t.leftChild = self.leftChild
            self.leftChild = t
    def insertRight(self, newNode):
        if self.rightChild == None:
            self.rightChild = BinaryTree(newNode)
        else:
            t = BinaryTree(newNode)
            t.rightChild = self.rightChild
            self.rightChild = t
    def getLeftChild(self):
        return self.leftChild
    def getRightChild(self):
        return self.rightChild
    def setRootValue(self, obj):
        self.key = obj
    def getRootValue(self):
        return self.key
    def __str__(self):
        return self.key

  2. Evaluate the expression using the parse tree.

Creating the parse tree
  1. Tokenise the expression.
  2. If the token is ( make a new node as the left child of the current node. Decend to the left child.
  3. If the current token is in the list ["+", "-", "/", "*"], set the root value of the current node to the operator represented by the current token. Add a right child to the current node and descend to the right.
  4. If the current token is a number, set the root value of the current node to the number and return to the parent.
  5. If the token is ) go to the parent of the current node.

String Operations

A string is a piece of text usually written using quotation marks. for example, "Hello World" is a string as well as "322324" and "$%£%$£^£^%£$$%£". However, 11 or True aren’t strings.

Strings are treated differently to numbers in programming languages. Take care not to use a string when you need to make calculations.
eg:

"6" * 4 → "6666"

6 * 4 → 24

Strings can also be concatenated.
eg in Python:

>>> a = "Hello"
>>> b = " world"
>>> c = a + b
>>> print(c)
'Hello world'

as well as split up:

>>> str = "HelloWorld"
>>> str[:3]
'Hel'
>>> str[2:4]
'll'
>>> str[5:]
'World'
>>> str[:]
'HelloWorld'

An example of data conversion in python:

>>> x = 6
>>> y = str(6)
>>> y
'6'
>>> z = int(y)
>>> z
6

In this example, x is declared as 6. It is then converted to a string and stored as y which is then printed implicitly. It could be printed explicitly with a print() statement. It is then converted back to an integer and stored as z which is then printed implicitly.
